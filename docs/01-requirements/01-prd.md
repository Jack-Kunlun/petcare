## 一、产品概述

### 1.1 产品定位

PetCare宠伴是一个连接宠物主人和服务提供者的O2O平台，提供上门宠物喂养、遛狗、陪玩等服务。平台采用**双模式接单机制**（悬赏大厅C2C + 平台定价B2C），结合**社区互动**和**标准化服务流程（SOP）**，打造可信赖的宠物托管服务生态。

### 1.2 目标用户

| 用户类型                     | 画像描述                                             | 核心需求                                   |
| ---------------------------- | ---------------------------------------------------- | ------------------------------------------ |
| **宠物主人**                 | 90后/00后都市白领，工作繁忙，节假日有出行需求        | 找到可靠的服务者照顾宠物，实时了解服务过程 |
| **实名宠物主（服务提供者）** | 有养宠经验的普通用户，愿意利用空闲时间提供服务       | 赚取额外收入，帮助其他宠物主，建立社交连接 |
| **认证宠托师（服务提供者）** | 有专业宠物护理经验的用户，希望全职或兼职从事宠物服务 | 获得稳定订单，提升专业技能，建立个人品牌   |
| **平台管理员**               | 平台运营团队                                         | 维护平台秩序，审核服务质量，监控运营数据   |

### 1.3 核心价值主张

- **对宠物主人**：便捷预约、透明服务、安全保障、社区交流
- **对服务提供者**：灵活接单、即时结算、技能提升、信用积累
- **对平台**：双向网络效应、数据驱动运营、标准化服务质量

## 二、产品架构

### 2.1 功能模块总览

```
PetCare宠伴
├── C端：宠物主人端（小程序/App）
│   ├── 账户管理（注册登录、个人资料、宠物档案）
│   ├── 服务预约（双模式：悬赏大厅/平台定价）
│   ├── 订单管理（查看进度、取消订单、评价服务）
│   ├── 社区互动（悬赏大厅、社区交流、宠物课堂）
│   └── 消息通知（订单通知、社区互动通知）
│
├── B端：服务提供者端（小程序/App）
│   ├── 身份认证（实名认证、认证宠托师升级）
│   ├── 订单处理（浏览订单、接单、执行SOP、完成服务）
│   ├── 收入管理（查看收入、提现、评分查看）
│   └── 信用记录（信用分、纠纷历史）
│
└── 后台管理系统（Web）
    ├── 用户管理（审核宠托师、处理投诉、信用管理）
    ├── 订单管理（订单监控、纠纷裁决、数据统计）
    ├── 内容审核（社区动态审核、宠物课堂管理）
    ├── RBAC权限管理（角色管理、权限分配、审计日志）
    └── 系统设置（SOP配置、评分阈值、费率设置）
```

### 2.2 技术架构要点

- **前端**：微信小程序（主）+ iOS/Android App（后续扩展）
- **后端**：NestJS 模块化单体。按用户、订单、支付结算、内容、权限等领域隔离；支付结算、通知和内容域达到拆分条件后再独立部署。
- **数据库**：PostgreSQL（业务主数据与事务）+ Redis（缓存、限流、BullMQ 异步任务）。日志先采用结构化日志聚合；未经容量和查询需求验证，不引入 MongoDB。
- **存储**：对象存储（图片/视频）+ CDN加速
- **第三方集成**：微信支付、微信登录、人脸识别、LBS定位

## 三、核心业务流程

### 3.1 双模式接单流程

详见《01-order-flow-diagram.md》文档，包含：

- 模式一：悬赏大厅接单流程（C2C）
- 模式二：平台定价接单流程（B2C）
- 两种模式对比和异常处理

### 3.2 标准化服务流程（SOP）

所有服务提供者必须严格执行5步SOP：

1. **进门消毒**（2分钟）：门口拍照→消毒→拍照确认
2. **拍照打卡**（3分钟）：宠物状态照片→记录精神状况
3. **执行服务**（根据类型）：喂食/遛狗/陪玩→过程拍照/视频
4. **清理现场**（3分钟）：清理垃圾→环境拍照
5. **离开拍照**（2分钟）：宠物离开照→门锁照→填写小结

**违规处罚**：

- 轻微违规（漏拍1-2张）：扣服务费10%
- 中度违规（漏拍3张以上或跳过步骤）：扣30%，评分-0.5
- 严重违规（完全未按SOP）：暂停7天，评分-1.0，重新培训

### 3.3 订单纠纷处理流程

详见《02-user-stories.md》第六章节，包含：

- 投诉发起→申诉回应→平台裁决→二次申诉→最终裁决
- 评分调整规则和信用记录管理

## 四、功能需求详述

### 4.1 C端：宠物主人端

#### 4.1.1 账户管理

**FR-001：微信一键登录**

- 支持微信授权登录，自动获取昵称和头像
- 首次登录需绑定手机号
- 登录成功后跳转首页

**FR-002：宠物档案管理**

- 可添加最多5只宠物
- 每只宠物需填写：名字、品种、年龄、体重、性别、绝育情况
- 可上传最多5张照片
- 记录特殊习性、过敏史、禁忌食物

#### 4.1.2 服务预约

**FR-003：发布悬赏订单（模式一）**

- 选择服务类型（喂养/遛狗/陪玩）
- 选择服务日期和时间段（精确到小时）
- 系统推荐价格区间（基于距离、时长、宠物数量）
- 宠物主自主设定悬赏金额（需在推荐区间内）
- 订单有效期48小时，超时自动关闭

**FR-004：预约平台定价服务（模式二）**

- 选择服务类型、时间、宠物
- 系统显示平台统一定价（不可修改）
- 提交后进入派单池，由认证宠托师抢单

**FR-005：确认/拒绝接单人**

- 有人接单后可查看接单人基本信息（昵称、头像、评分、认证标识）
- 可查看历史评价（不显示具体订单信息）
- 点击"确认"锁定订单，点击"拒绝"重新开放
- 同一订单最多3人意向接单

**FR-006：打回宠托师（仅模式二）**

- 最多可打回3次
- 第4次自动匹配当前接单人，不可再打回

#### 4.1.3 订单管理

**FR-007：查看服务过程**

- 实时查看服务提供者上传的照片/视频（带时间水印）
- 查看SOP执行进度（5个步骤）
- 查看服务开始和结束时间

**FR-008：取消订单**

- 服务前24小时取消：全额退款
- 服务前2-24小时取消：扣除30%违约金
- 服务前2小时内：不可取消（需联系客服）

**FR-009：评价服务**

- 服务完成后24小时内可评价
- 评分维度：服务态度、专业程度、准时性、SOP执行、整体满意度（1-5星）
- 可上传照片/视频佐证，填写文字评价（最多500字）
- 评分低于3星需人工复核

#### 4.1.4 社区互动

**FR-010：悬赏大厅**

- 基于LBS展示附近悬赏订单（默认5公里）
- 列表显示：服务类型、时间、地点（模糊）、金额、宠物类型
- 仅实名认证用户可见接单按钮

**FR-011：社区交流**

- 发布动态：照片（最多9张）或视频（最长60秒）+文字（最多1000字）
- 互动功能：点赞、评论（最多200字）、转发
- 关注其他用户，查看其动态

**FR-012：宠物课堂**

- 官方发布专业养宠知识文章和视频
- 分类：喂养指南、健康管理、行为训练、疾病预防
- 支持搜索、收藏、评论提问

---

### 4.2 B端：服务提供者端

#### 4.2.1 身份认证

**FR-013：实名认证（必需）**

- 上传身份证正反面
- 人脸识别验证
- 签署服务协议
- 认证有效期1年

**FR-014：申请认证宠托师（可选）**

- 填写基本资料、从业经验
- 授权查询微信支付分（需≥650）
- 通过线上培训考试（80分及格）
- 审核通过后获得"认证宠托师"标识

#### 4.2.2 订单处理

**FR-015：浏览悬赏订单（实名宠物主）**

- 基于LBS展示附近订单
- 查看订单详情（宠物信息、特殊要求）
- 点击"我要接单"提交意向

**FR-016：接收平台订单通知（认证宠托师）**

- 系统推送附近订单（最多5个）
- 15分钟内需响应，超时自动释放
- 可接受或拒绝（拒绝率>50%降低优先级）

**FR-017：执行SOP**

- 严格按5步流程执行，每步需拍照/录像
- 用户可实时查看上传内容
- 未完成SOP无法标记服务完成

#### 4.2.3 收入管理

**FR-018：查看收入明细**

- 显示今日/本周/本月/累计收入
- 区分已结算和待结算
- 纠纷中订单显示"冻结"状态

**FR-019：提现**

- 支持提现到微信零钱
- 最低10元，T+0实时到账
- 每月免费提现3次，超出每次1元手续费

**FR-020：查看评分和评价**

- 显示综合评分（1-5星，保留1位小数）
- 显示各维度平均分
- 评分<3.5显示警告，<3.0暂停接单需重新培训

#### 4.2.4 信用记录

**FR-021：查看信用记录**

- 显示信用分数（默认100分）
- 显示纠纷统计和最近10条记录
- 信用分<80警告，<60限制功能，<40暂停账号

### 4.3 后台管理系统

#### 4.3.1 用户管理

**FR-022：审核认证宠托师**

- 查看申请人资料、证件、微信支付分、考试成绩
- 通过或驳回（需填写原因）

**FR-023：处理投诉**

- 查看双方证据（SOP记录、照片、GPS、聊天记录）
- 裁决选项：支持宠物主/支持服务提供者/双方责任/证据不足
- 自动执行退款/结算/扣分

**FR-024：管理评分阈值**

- 设置暂停接单阈值（默认3.0星）
- 设置警告阈值（默认3.5星）
- 查看被暂停用户列表

#### 4.3.2 订单管理

**FR-025：订单监控**

- 查看所有订单列表和状态
- 筛选：时间范围、订单类型、状态
- 导出订单数据（Excel）

**FR-026：纠纷裁决**

- 初级客服72小时内初步裁决
- 高级客服处理二次申诉（72小时内最终裁决）
- 自动更新双方信用记录

#### 4.3.3 内容审核

**FR-027：审核社区动态**

- 系统自动识别疑似违规内容
- 人工复核，可删除并通知发布者
- 多次违规限制发布权限

**FR-028：管理宠物课堂**

- 发布文章和视频
- 设置分类、标签、置顶
- 查看阅读量和互动数据

#### 4.3.4 RBAC权限管理

**FR-029：角色管理**

- 创建自定义角色，预设系统角色
- 角色启用/禁用、复制、删除保护

**FR-030：权限点管理（按钮级别）**

- 定义细粒度权限点（菜单/按钮/API）
- 按模块组织：用户、订单、内容、财务、商家等
- 支持批量导入/导出

**FR-031：角色权限分配**

- 可视化权限树，勾选分配
- 支持全选/全不选
- 权限对比功能

**FR-032：用户角色分配**

- 支持多角色分配，权限取并集
- 批量分配，移除保护

**FR-033：前端按钮级权限控制**

- 使用权限指令控制按钮显示/隐藏
- API层双重校验
- 提供调试模式

**FR-034：商家端权限隔离（预留）**

- 预留"商家管理"模块权限
- 数据权限与功能权限分离
- 按商家ID过滤数据

**FR-035：权限审计日志**

- 记录所有权限变更操作
- 不可删除，只可归档
- 敏感操作需额外审批

#### 4.3.5 数据统计

**FR-036：运营数据看板**

- 日活/月活用户数
- 订单量、成交金额、转化率
- 区分悬赏订单和平台订单
- 服务提供者数量、活跃度、平均评分
- 纠纷统计：投诉率、裁决分布、平均处理时长
- 支持按时间范围筛选，数据实时更新（延迟≤5分钟）

## 五、非功能性需求

### 5.1 性能要求

| 指标               | 要求    |
| ------------------ | ------- |
| 首页加载时间       | < 2秒   |
| 订单创建响应时间   | < 1秒   |
| 图片上传压缩后单张 | < 2MB   |
| 并发用户数支持     | ≥ 10000 |
| API响应时间（P95） | < 500ms |

### 5.2 安全要求

- 用户敏感信息（身份证、手机号）加密存储（AES-256）
- 支付接口采用HTTPS加密传输
- 防止SQL注入、XSS攻击、CSRF攻击
- 定期备份数据库（每日全量备份，保留30天）
- 实名认证数据对接公安系统验证
- RBAC权限校验：所有API接口需进行权限验证
- 操作日志记录：关键操作记录用户、时间、IP、操作内容

### 5.3 兼容性要求

- 微信小程序基础库版本 ≥ 2.10.0
- 支持iOS 12+ 和 Android 8.0+
- 适配主流屏幕尺寸（375x667 ~ 414x896）
- 深色模式适配（可选）

### 5.4 可用性要求

- 系统可用性 ≥ 99.9%
- 支持7×24小时服务
- 故障恢复时间 < 30分钟
- 数据一致性：订单状态、支付状态强一致

### 5.5 可扩展性要求

- 模块化单体支持 API 与 Worker 的独立水平扩展；按领域边界演进为独立服务，不以拆分服务作为 MVP 前置条件
- RBAC权限体系预留商家端扩展能力
- 订单、支付状态通过 PostgreSQL 事务和条件更新保障强一致；金额以最小货币单位整数或 Decimal 存储
- 采用 Transactional Outbox + BullMQ 异步处理通知、结算、超时关闭订单；消费者必须幂等
- 当订单数据量、读写负载或合规边界达到阈值时，再进行分区、读副本或按领域的数据拆分

## 七、接口规范（核心API）

### 7.1 认证与授权

| 接口                   | 方法 | 说明       | 权限要求 |
| ---------------------- | ---- | ---------- | -------- |
| /api/auth/wechat-login | POST | 微信登录   | 无       |
| /api/auth/bind-phone   | POST | 绑定手机号 | 已登录   |

## 九、里程碑规划

### Phase 1：MVP（预计2个月）

**目标**：实现核心闭环，支持小规模内测

**功能范围**：

- C端：注册登录、宠物档案、发布悬赏订单、预约平台服务、订单管理、评价
- B端：实名认证、浏览订单、接单、执行SOP、完成服务、查看收入
- 后台：用户审核、订单监控、基础数据统计

**验收标准**：

- 支持100个种子用户完成完整服务流程
- 订单创建到完成全流程跑通
- SOP执行和评价功能正常

---

### Phase 2：第一版迭代（预计1.5个月）

**目标**：完善社区功能和权限体系

**功能范围**：

- C端：社区交流（发布动态、点赞评论）、宠物课堂、消息通知
- B端：认证宠托师升级、评分查看、信用记录
- 后台：RBAC权限管理（角色、权限、分配、前端控制）、内容审核、纠纷处理基础

**验收标准**：

- 社区UGC内容日均100+条
- 后台权限体系完整，支持多角色管理
- 纠纷处理流程跑通

---

### Phase 3：优化与扩展（预计2个月）

**目标**：提升用户体验，准备商家端扩展

**功能范围**：

- C端：关注功能、动态转发、搜索优化
- B端：收入报表导出、申诉功能
- 后台：二次申诉、信用管理、商家端权限预留、高级数据统计
- 技术：性能优化、监控告警、容灾备份

**验收标准**：

- 系统可用性≥99.9%
- 支持10000并发用户
- 商家端权限体系可无缝扩展

## 十、风险与应对

| 风险项                                 | 影响程度 | 应对措施                                                    |
| -------------------------------------- | -------- | ----------------------------------------------------------- |
| **服务安全问题**（宠物受伤、物品损坏） | 高       | 严格SOP执行+照片证据+保险机制+纠纷快速处理                  |
| **服务提供者供给不足**                 | 高       | 早期补贴激励+简化认证流程+口碑传播                          |
| **恶意投诉/刷单**                      | 中       | 信用记录体系+异常行为监控+人工复核                          |
| **隐私泄露**                           | 高       | 数据加密+权限控制+审计日志+合规审查                         |
| **支付纠纷**                           | 中       | 平台托管+清晰退款规则+快速仲裁                              |
| **技术扩展性**                         | 中       | 模块边界+Outbox 异步任务+容量压测；按指标拆分服务或数据存储 |
